<중첩 클래스의 접근 제한> 		chap09/sec01/Car 참조

- 멤버 클래스가 인스턴스냐 정적이냐에 따라 바깥 클래스의 필드와 메소드에 접근 제한 생김
- 인스턴스 멤버 클래스 : 바깥 클래스의 모든 필드와 메소드에 접근 O
- 정적 멤버 클래스 : 바깥 클래스의 정적 필드와 정적 메소드에만 접근 O, 인스턴스 필드와 메소드에는 접근 X


<중첩 클래스에서 바깥 클래스 참조 얻기 >

- 클래스 내부에서 this는 객체 자신의 참조
- 중첩 클래스에서 this 사용시 바깥 클래스의 객체 참조가 아니라 중첩 클래스의 객체 참조가 됨
- 중첩 클래스 내부에서 바깥 클래스의 객체 참조 얻으려면 '바깥 클래스.this.' 사용함
  예 ) Car.this.speed      /          Car.this.speedUp()


<중첩 인터페이스>

- 인스턴스 멤버 인터페이스, 정적 멤버 인터페이스 있음

1. 인스턴스 멤버 인터페이스 : 바깥 클래스의 객체가 있어야 사용 가능
2. 정적 멤버 인터페이스 : 바깥 클래스의 객체 없이 바깥 클래스만으로 바로 접근 가능

<자바스크립트>
- 0, null, ""(빈문자), undefined는 조건식에 입력되면 false로 반환,
 입력값이 있으면 항상 true

- prompt 입력 받을 때는 기본적으로 문자열로 인식, 정수로 바꾸고 싶으면 parseInt 이용해야 함
  예)  var num1 = parseInt(prompt("첫번째 숫자를 입력하세요"));

- confirm(" ");   : boolean 타입, true와 false 결과값으로 나옴
