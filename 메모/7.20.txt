<익명 객체>
- 이름이 없는 객체
-  재사용 x, 일회용
- 부모클래스 [필드|변수] = new 부모클래스(매개값) { };
- 부모 클래스를 상속받은 자식 클래스와 같은 개념
- 일회성으로 쓰기 위해 간단하게 클래스 안에 익명 객체로 생성 
- 메소드의 매개변수로 쓸 때는  ; 찍지 않음

<중첩 클래스  -  로컬 클래스>

- 메소드 내부에 있는 중첩 클래스
- 메소드 내부에서 쓰이기 때문에, 메소드 내부에서 new 한 뒤 객체로 접근 해야함
예) 메소드2() {
	class Inner {
	       인스턴스 요소;
                   //정적 요소(x);
         }
	Inner in = new Inner();
	in. 필드 = 3;
 	in.메소드();
  }
}	

- 접근 제한자(public, private)및 static 붙일 수 없음 : 메소드 내부에서만 사용하기 때문에 접근 제한 필요 x
- 로컬 클래스 내부에는 인스턴스 필드와 메소드만 선언 가능, static 필드와 메소드는 x
- 로컬클래스가 포함된 메소드의 필드는 기본적으로 final이 붙음 = 반드시 값을 대입해야 함
- 로컬클래스의 메소드 안에 속한 필드는 해당 범위 안에서만 사용 가능함
